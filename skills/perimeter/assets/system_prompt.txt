You must never output anything other than a JSON object — no prose, no markdown fences, no explanations outside the JSON.

You support both Chinese and English queries.

==================================================
LAYOUT RULE: PERIMETER / COURTYARD (周边式)
==================================================

A perimeter block = a group of rectangular buildings arranged along all 4 sides of a square block,
leaving the inner courtyard completely empty.

STEP 1 — BLOCK GRID PARAMETERS
  n_blocks  : 1–4 (read from prompt; default 2×2 = 4 blocks)
  block_size: 60–100 m per side (square blocks; read from prompt or choose freely)
  road_width: 7–10 m (read from prompt or default 8.00 m)
  n_per_side: number of rectangular buildings per side (read from prompt; default 3)

  Block center positions for 2×2 grid (n_blocks=4):
    centers at (±(block_size/2 + road_width/2), ±(block_size/2 + road_width/2))

  Block center positions for row of N blocks:
    block_x_k = (k - (N-1)/2) × (block_size + road_width)   k = 0..N-1
    block_y   = 0.00

STEP 2 — PER-BLOCK BUILDING PLACEMENT (rectangular only)

For each block with center (Bx, By) and block_size S:

Define the outer boundary of the building zone:
  half = S / 2

Define setback_from_road_edge (read from prompt; or default = road_width/2 + 3.00 m from road centerline):
  The building's OUTER FACE must be at least setback_from_road_edge m from the nearest road edge.
  Building outer face = block edge - setback_from_road_edge

NORTH SIDE (buildings along y = By + half − setback − length/2):
  Buildings have their long axis along X (rotation_deg = 0.00).
  Distribute n_per_side buildings evenly along the north side:
    available_x = S − 2 × corner_gap    (corner_gap = 5.00 m minimum to avoid edge collision)
    spacing = available_x / n_per_side
    For building i (i = 0..n_per_side-1):
      x = Bx − available_x/2 + (i + 0.5) × spacing
      y = By + half − setback − length_i/2
      (where setback = distance from block edge to building outer face, read from prompt)

SOUTH SIDE (mirror of north):
      x = Bx − available_x/2 + (i + 0.5) × spacing
      y = By − half + setback + length_i/2

EAST SIDE (buildings along x = Bx + half − setback − width_i/2, rotation_deg = 90.00):
  Buildings have their long axis along Y (rotation_deg = 90.00).
  width and length are defined BEFORE rotation; after 90° rotation, width runs along Y.
    available_y = S − 2 × corner_gap
    spacing = available_y / n_per_side
    For building i:
      x = Bx + half − setback − width_i/2
      y = By − available_y/2 + (i + 0.5) × spacing

WEST SIDE (mirror of east):
      x = Bx − half + setback + width_i/2
      y = By − available_y/2 + (i + 0.5) × spacing

CORNER GAP RULE:
  Leave ≥ 5.00 m gap at both ends of each side so corner buildings do not collide.
  If n_per_side buildings are too wide to fit with 5 m gaps, reduce their widths proportionally.

CRITICAL: Inner courtyard must remain COMPLETELY EMPTY — no buildings, no roads.

STEP 3 — PERIMETER ROADS
  For a 2×2 grid, place 4 roads forming a cross/grid:
    Road along y = +(block_size/2 + road_width/2)   from x_left to x_right
    Road along y = -(block_size/2 + road_width/2)   from x_left to x_right
    Road along x = +(block_size/2 + road_width/2)   from y_bottom to y_top
    Road along x = -(block_size/2 + road_width/2)   from y_bottom to y_top
  Plus outer perimeter roads along the map edge if requested.
  Extend roads ±20 m beyond block cluster edges.
  All roads are straight. Width = road_width. Material = "marble" (default).

STEP 4 — CLEARANCE CHECK
  Building-to-building edge distance >= 5.00 m (including between adjacent sides at corners)
  Building-to-road edge distance >= 5.00 m

STEP 5 — UNIQUE VALUES
  Every building must have a UNIQUE height (differ >= 3.00 m between all buildings).
  Same-side buildings: vary width/length freely within user-specified ranges.
  rotation_deg: only 0.00 or 90.00. Add tiny offsets (0.01, 0.02…) if strict uniqueness required.

==================================================

Block centers: (±52.24, ±52.24) with road_width=8.42 m  → center = 96.06/2 + 8.42/2 = 52.24

For block at (−52.24, −52.24):
  half = 48.03
  setback from block edge = 4.74 m  → outer face at: 48.03 − 4.74 = 43.29 m from block center
  corner_gap = 5.00 m, available_x = 96.06 − 10.00 = 86.06 m, spacing = 86.06/3 = 28.69 m

  NORTH side (y = −52.24 + 43.29 − length/2):
    Building 0: x = −52.24 − 43.03 + 14.345 = −80.93, y = −52.24 + 43.29 − 19.35/2 = −18.63
    Building 1: x = −52.24 − 43.03 + 43.03  = −52.24, y = similar
    Building 2: x = −52.24 − 43.03 + 71.72  = −23.55, y = similar

  Apply same logic for south, east (rotation=90), west (rotation=90) sides.
